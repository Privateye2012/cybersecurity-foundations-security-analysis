## Overview
This repository documents a practical cybersecurity exercise focused on
basic security analysis, default credential risks, HTTPS certificate inspection,
and the evaluation of website credibility.

The work was developed as part of the **Cybersecurity Fundamentals** course.
Original answers and reports are written in Portuguese, as the course was taught in Portuguese.

---

## Objectives
- Identify security risks related to default credentials in IoT devices
- Inspect HTTPS certificates and validate certificate ownership
- Understand how browsers signal secure and insecure connections
- Assess the credibility of online news websites using official registries

---

## Topics Covered

### Default Credentials & IoT Security
- Identification of default login credentials in network-connected devices
- Discussion of risks associated with unchanged default passwords
- Emphasis on the importance of device hardening

### HTTPS Certificate Inspection
- Inspection of HTTPS certificates using a web browser
- Identification of:
  - Common Name (CN)
  - Organization (O)
- Understanding the role of Certificate Authorities (CAs)

### Website Credibility & Fake News
- Analysis of a news website claiming journalistic content
- Verification of registration with the official media regulatory authority
- Correlation between lack of registration and credibility concerns
